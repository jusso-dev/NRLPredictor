## 1. Scraper Health

### Architecture note

There is **no `app/Scrapers/` directory**, no `BaseScraper` class, no `ScraperCache` model, and no `scraper_runs` table. All scraping is done via queued Jobs in `app/Jobs/` using `App\Support\HttpScraper` (a rate-limited HTTP client). Fetch history is logged to the `data_fetch_logs` table via the `LogsDataFetch` trait.

All show `status=success` with `records_updated=0`. This is misleading — jobs return "success" even when they parse nothing. No actual failures are being logged because the scrapers fail silently (they return 0 when CSS selectors match nothing).

### Parse failures

No explicit parse failures in logs. The failure mode is **silent zero results** — the HTML scrapers hit a page, fail to match any CSS selectors, and return 0 records without logging an error.

No hardcoded year values. All jobs use `now()->year` or `$round->season` dynamically. The URL patterns are season-agnostic.

4. **Scorer position mismatch**: 325 of 459 try events have player position = "" (empty). Scorer data from the match JSON doesn't include position, and 329 of 505 players have no position set.

## 5. Prompt Quality

### Location

The prompt is NOT in `resources/prompts/try-predictor.md` — that directory does not exist. The prompt is inline in `python-agent/agent.py` as `SYSTEM_PROMPT` (line 20-35).

### Current prompt text

```
You are a professional NRL rugby league analyst reviewing try-scorer predictions.

For each match:
1. Call get_match_context to understand team lists, injuries, venue, and kickoff.
2. Call get_top_predictions to see the statistically-predicted top 10 try scorers with their signals.
3. For interesting candidates, call get_player_deep_stats to inspect career, venue, and head-to-head records.
4. Call get_team_articles for both teams to catch narrative factors (coach comments, motivation).
5. Identify compound advantages where multiple signals stack (e.g. hot form + weak opposing edge + milestone).
6. For each of the top ~8 players, call submit_adjusted_prediction with:
   - adjusted_score: the statistical score +/- up to 15 points based on contextual factors.
   - reasoning: 2-3 sentences citing specific stats and context.

Be decisive. Favour wingers and fullbacks facing disrupted defensive edges. Penalise predictions that depend on signals
already contradicted by the news (e.g. a "returning player" whom the coach has flagged as cautious). Cite concrete
numbers whenever you reference a signal.
```
